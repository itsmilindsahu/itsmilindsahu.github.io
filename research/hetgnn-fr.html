Overview

What is HETGNN-FR?

HETGNN-FR is a fraud ring detection system that models financial transaction networks as heterogeneous graphs — graphs where nodes and edges have distinct types. A fraud ring is a coordinated group of accounts acting together to commit financial fraud, which is invisible to individual-account anomaly detectors but emerges as a structural pattern in the transaction graph.

KEY INSIGHT: Modern financial fraud is coordinated via networks of accounts sharing devices, IPs, and target merchants. A single-node classification approach fails because individual accounts mimic benign behavior; fraud detection must be modeled as a heterogeneous graph search for community rings.

Heterogeneous Graph Transformer (HGT)

HGT uses type-specific attention mechanisms — separate attention heads for each (source type, edge type, target type) triplet. This allows the model to learn how information flows differently across different relationship types (e.g., "account shares device with account" vs. "account transacts at merchant").

Time2Vec Temporal Encoding

Financial fraud rings often exhibit characteristic temporal patterns — coordinated activity within a short window. Time2Vec encodes transaction timestamps as a mix of linear and sinusoidal components, giving the model a learnable representation of time that captures both trend and periodic structure.

Implementation

Technical Details & Code

HETGNN-FR is built using PyTorch Geometric. Below is the implementation snippet for the Time2Vec temporal feature embedding and the HGT convolutional framework definition:

import torch
import torch.nn as nn
from torch_geometric.nn import HGTConv

class Time2Vec(nn.Module):
    def __init__(self, in_features=1, out_features=16):
        super(Time2Vec, self).__init__()
        self.out_features = out_features
        # Linear projection parameters
        self.w0 = nn.Parameter(torch.randn(in_features, 1))
        self.b0 = nn.Parameter(torch.randn(1))
        # Periodic projection parameters
        self.w = nn.Parameter(torch.randn(in_features, out_features - 1))
        self.b = nn.Parameter(torch.randn(out_features - 1))

    def forward(self, t):
        # Linear component
        v0 = torch.matmul(t, self.w0) + self.b0
        # Periodic (sinusoidal) component
        v = torch.sin(torch.matmul(t, self.w) + self.b)
        return torch.cat([v0, v], dim=-1)

class HeteroFraudGNN(torch.nn.Module):
    def __init__(self, metadata, hidden_channels, out_channels, num_heads=4):
        super().__init__()
        self.t2v = Time2Vec(in_features=1, out_features=16)
        self.convs = torch.nn.ModuleList()
        # Initialize multi-relational Heterogeneous Graph Transformer layers
        self.convs.append(HGTConv(hidden_channels, hidden_channels, metadata, num_heads))
        self.convs.append(HGTConv(hidden_channels, hidden_channels, metadata, num_heads))
        self.lin = torch.nn.Linear(hidden_channels, out_channels)

    def forward(self, x_dict, edge_index_dict):
        # Propagate embeddings through relational convolution layers
        for conv in self.convs:
            x_dict = conv(x_dict, edge_index_dict)
        return {key: self.lin(x) for key, x in x_dict.items()}
