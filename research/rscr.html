Abstract

The Problem with Standard Survival Models

Standard survival models applied to mortgage credit risk (Cox PH, DeepHit, XGBoost-AFT) predict time-to-default but systematically ignore two structural properties of credit portfolios that are well-established in financial theory:

KEY ADVANCEMENT: Standard survival models ignore portfolio-level logic (e.g., macroeconomic vintage ordering and expected residual lifetimes). RRSM bridges the gap between deep survival learning and financial theory by introducing structural regularizers directly into the loss function optimization.

PROPERTY 01
Cohort Hazard Ordering
Loans originated during adverse macroeconomic conditions (e.g., 2008) should have monotonically higher hazard rates than loans from better vintages. Standard models don't enforce this structural constraint.
PROPERTY 02
Mean Residual Lifetime (MRL)
For prepayment risk, the expected remaining loan life at each time point matters as much as the instantaneous hazard rate. Ignored by most survival ML approaches despite its importance in mortgage servicing.

Methodology

The RRSM Loss Function

The Reliability-Regularized Survival Model (RRSM) adds both missing properties as regularization terms to the standard survival loss:

L_RRSM = L_survival
       + λ₁ · L_hazard_ordering  ← cohort monotonicity
       + λ₂ · L_MRL                ← prepayment risk term

L_hazard_ordering

A pairwise penalty that enforces the constraint: for any two loan cohorts where cohort A has worse macroeconomic origination conditions than cohort B, the predicted hazard function for cohort A must be pointwise ≥ the hazard for cohort B at all time points. Implemented as a soft hinge loss over cohort pairs.

L_MRL

Derived from the survival function S(t) as the integral of S from t to ∞, normalized by S(t). The MRL regularizer encourages the model's estimated survival function to produce MRL curves consistent with historical portfolio behavior, providing a more complete picture of prepayment risk than instantaneous hazard alone.

Competing Risks Framing

The full model treats default and prepayment as competing risks — two causes of loan termination that interact. A loan can only experience one event, so standard single-event survival models misspecify the problem. RRSM uses a cause-specific hazard approach that jointly models both risks with shared covariate representations.

Implementation

Custom PyTorch Regularizer

Below is the mathematical execution snippet for calculating the vintage hazard-ordering soft-hinge penalty and the integrated cohort Mean Residual Lifetime regularizer:

import torch
import torch.nn as nn

class RRSMLoss(nn.Module):
    def __init__(self, lambda_1=0.2, lambda_2=0.1):
        super().__init__()
        self.lambda_1 = lambda_1 # Weight for vintage ordering
        self.lambda_2 = lambda_2 # Weight for MRL calibration

    def forward(self, pred_hazard, target_time, target_event, macro_rank):
        # 1. Base log-likelihood survival loss (censored Cox/DeepHit variant)
        loss_surv = cox_likelihood(pred_hazard, target_time, target_event)

        # 2. Pairwise Cohort Hazard-Ordering Monotonicity Hinge Penalty
        n = pred_hazard.size(0)
        loss_ordering = torch.tensor(0.0, device=pred_hazard.device)
        pairs = 0
        for i in range(n):
            for j in range(n):
                if macro_rank[i] > macro_rank[j]: # cohort i orig in worse conditions
                    # Enforce hazard(i) >= hazard(j), penalize if hazard(i) < hazard(j)
                    loss_ordering += torch.clamp(pred_hazard[j] - pred_hazard[i], min=0.0)
                    pairs += 1
        if pairs > 0:
            loss_ordering = loss_ordering / pairs

        # 3. Mean Residual Lifetime Calibration Regularizer
        # MRL(t) = \int_t^\infty S(u) du / S(t)
        pred_survival = torch.exp(-torch.cumsum(pred_hazard, dim=1))
        pred_mrl = torch.sum(pred_survival, dim=1) # Integral approximation
        loss_mrl = torch.mean((pred_mrl - target_time) ** 2)

        return loss_surv + self.lambda_1 * loss_ordering + self.lambda_2 * loss_mrl
